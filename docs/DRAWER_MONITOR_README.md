# 抽屜閉合監測應用程式使用說明

## 概述

這是一個使用 MN96100C 2.5D 影像感測器來監測抽屜閉合狀態的調校工具。應用程式提供實時深度圖像顯示、時間序列分析和閾值調整功能。

## 感測器原理

- **MN96100C 2.5D Sensor**: 輸出 160x160 的 8-bit 灰度圖像
- **測量方式**: 偵測反射光強度（晶片內部已扣除環境光）
- **深度關係**: 反射光強度的開平方根後的倒數大約正比於距離
- **相對深度**: 輸出為相對深度資訊，非絕對距離
- **影響因素**: 反射面材質、主動光源波長和強度

## 應用場景

### 抽屜閉合監測
1. **安裝位置**: 2.5D camera 直對抽屜平面
2. **未閉合狀態**: 相機直接看到更深的地板（較低的像素值）
3. **閉合過程**: 抽屜逐漸插入，遮擋地板（像素值逐漸上升）
4. **完全閉合**: 地板完全被遮住（較高的像素值）

## 功能特色

### 1. 實時圖像顯示
- 480x480 放大顯示（原始 160x160）
- 偽彩色深度圖（JET colormap）
- ROI 區域框線標示

### 2. 相機參數控制
可調整的超參數：
- **幀率 (Frame Rate)**: 
  - FULL (1/1 全速)
  - HALF (1/2)
  - QUARTER (1/4, 預設)
  - EIGHTH (1/8)
  - SIXTEENTH (1/16)
  
- **LED 電流 (LED Current)**:
  - LOW (50mA × 2)
  - MEDIUM (100mA × 2)
  - HIGH (200mA × 2)
  - ULTRA_HIGH (400mA × 2, 預設)

- **曝光設定 (Exposure Setting)**:
  - DEFAULT (預設)
  - UNKNOWN (替代設定)

### 3. ROI 區域設定
- 可選擇性啟用感興趣區域（Region of Interest）
- 設定 X1, Y1（左上角）和 X2, Y2（右下角）
- 範圍: 0-160 像素
- 預設 ROI: (40, 40) 到 (120, 120)

### 4. 時間序列分析
實時顯示：
- 深度指標（平均像素值）時間序列曲線
- 開啟閾值參考線（紅色虛線）
- 閉合閾值參考線（綠色虛線）
- 最多保存 500 幀歷史數據

### 5. 閾值設定
- **開啟狀態閾值**: 低於此值判定為"完全開啟"（預設: 50）
- **閉合狀態閾值**: 高於此值判定為"完全閉合"（預設: 120）
- 介於兩者之間為"閉合中"

### 6. 狀態顯示
- 抽屜狀態：完全開啟（紅）/ 閉合中（橙）/ 完全閉合（綠）
- 深度指標數值
- 平均像素值
- 實時 FPS

### 7. 設定管理
- 儲存/載入所有參數設定
- 設定檔: `drawer_monitor_settings.json`

## 使用步驟

### 1. 安裝依賴
```bash
pip install opencv-python numpy pillow matplotlib
pip install pyusb
```

### 2. 啟動應用程式
```bash
python drawer_monitor.py
```

### 3. 初次設定流程

#### 步驟 A: 相機參數調整
1. 根據環境光線條件調整 **LED 電流**
   - 明亮環境 → MEDIUM 或 LOW
   - 暗環境 → HIGH 或 ULTRA_HIGH
   
2. 根據需要的採樣率調整 **幀率**
   - 快速運動監測 → HALF 或 FULL
   - 一般監測 → QUARTER（推薦）
   - 省電模式 → EIGHTH 或 SIXTEENTH

3. 點擊「套用參數」並「啟動相機」

#### 步驟 B: ROI 設定
1. 勾選「啟用 ROI」
2. 根據實際安裝位置調整 ROI 範圍
   - 建議只監測抽屜會遮擋的區域
   - 避免包含不會改變的背景區域
3. 點擊「套用 ROI」

#### 步驟 C: 閾值校準
1. **測試完全開啟狀態**:
   - 將抽屜完全拉開
   - 觀察「深度指標」數值（應該較低，例如 30-60）
   - 記錄這個範圍

2. **測試完全閉合狀態**:
   - 將抽屜完全推入閉合
   - 觀察「深度指標」數值（應該較高，例如 120-200）
   - 記錄這個範圍

3. **設定閾值**:
   - 開啟狀態閾值 = 開啟狀態的最大值 + 10
   - 閉合狀態閾值 = 閉合狀態的最小值 - 10
   - 點擊「套用閾值」

4. **驗證**:
   - 多次開關抽屜
   - 觀察時間序列圖和狀態顯示是否正確
   - 微調閾值直到滿意

#### 步驟 D: 儲存設定
1. 點擊「儲存設定」
2. 設定將保存到 `drawer_monitor_settings.json`
3. 下次啟動時自動載入

### 4. 日常使用
1. 啟動應用程式（自動載入上次設定）
2. 點擊「啟動相機」
3. 觀察抽屜狀態和時間序列
4. 用完後點擊「停止相機」

## 優化建議

### 提高穩定性
1. **LED 電流**: 
   - 較高的 LED 電流提供更強的信號
   - 但會增加功耗和發熱
   - 建議先用 ULTRA_HIGH 測試，再降低到能穩定工作的最低值

2. **幀率**:
   - 較低的幀率可以減少雜訊
   - QUARTER 是穩定性和響應速度的良好平衡點

3. **ROI**:
   - 使用 ROI 可以排除干擾區域
   - 只監測關鍵區域可提高判斷準確性

### 抗干擾措施
1. 固定相機與抽屜的相對位置
2. 避免外部光源直射感測器
3. 確保抽屜表面平整，材質均勻
4. 定期清潔感測器鏡頭

### 閾值調整技巧
1. 觀察時間序列圖找出穩定區間
2. 設定足夠的安全裕度（hysteresis）
3. 考慮使用中位數或百分位數而非單純平均值
4. 記錄不同溫度/濕度下的數值變化

## 故障排除

### 問題: 相機無法啟動
- 檢查 USB 連接
- 確認 VID/PID 是否正確（預設 0x04F3:0x0C7E）
- 檢查 libusb 驅動是否安裝

### 問題: 圖像噪點多
- 提高 LED 電流
- 降低幀率
- 調整曝光設定

### 問題: 狀態判斷不準確
- 重新校準閾值
- 調整 ROI 範圍
- 檢查安裝位置是否穩定

### 問題: FPS 過低
- 提高幀率設定
- 檢查 CPU 使用率
- 確保 USB 2.0/3.0 連接正常

## 技術細節

### 深度計算原理
```
像素值 (8-bit) = 反射光強度
相對距離 ∝ 1 / √(反射光強度)
```

因此：
- **高像素值** → 強反射 → 近距離（抽屜插入）
- **低像素值** → 弱反射 → 遠距離（看到地板）

### 數據流程
```
1. 相機捕獲 160x160 原始數據
2. 選擇 ROI 或全圖
3. 計算統計指標（平均、最小、最大、標準差）
4. 深度指標 = 平均像素值
5. 與閾值比較判斷狀態
6. 更新 UI 和時間序列
```
